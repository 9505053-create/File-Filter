# File Filter（資料夾即時篩選器 / FolderFilter）

> 一個輕量的 Windows 資料夾即時篩選工具，用 **AutoHotkey v2** 開發。
> 在你「當下開著的檔案總管資料夾」按一個熱鍵，就跳出篩選視窗，打字即時過濾檔名 —— 像 QTTabBar 的篩選功能，但**不注入檔案總管，所以 Windows 更新或重開機都不會壞**。

---

## 為什麼做這個

- **QTTabBar 的痛點**：它是 shell extension（注入 `explorer.exe`），Win11 每次更新就容易失效，得反覆改登錄檔重新啟用，很不穩定。
- **Everything 的痛點**：很快，但預設搜尋整台電腦，要限定當前資料夾每次都得手動設定，不夠順手。
- **這支工具的定位**：綁定「你現在正在看的那個資料夾」做即時篩選，速度快、版面乾淨，而且因為**完全不注入檔案總管**（只用穩定的 `Shell.Application` COM 讀取當前路徑），更新／重開機都不會把它弄壞。

---

## 功能

### 篩選 / 搜尋
- **即時過濾**：打字當下就過濾，不需等索引。
- **多關鍵字**：空白分隔，AND 條件（例：`報告 2024`）。
- **萬用字元**：關鍵字含 `*` 或 `?` 時視為樣式比對（例：`*.md`、`報告*`、`?.txt`）。
- **行內排除**：`-關鍵字` 可排除特定字（例：`報告 -舊`）。
- **含子資料夾**：可勾選遞迴往下找，名稱欄會顯示相對路徑。

### 排除條件（可勾選，即時生效）
- 排除資料夾 / 排除檔案
- 排除小於 N KB 的檔案 / 排除大於 N KB 的檔案
- 排除 N 天前的檔案（藏舊檔）/ 排除 N 天以後的檔案（藏新檔）
- 兩個天數條件可組合出時間區間

### 清單與欄位
- 欄位：**名稱 / 副檔名 / 大小 / 類型 / 修改日期 / 建立日期**
- 點欄位標題排序（大小依實際位元組、日期依實際時間，再點一次反向）
- **自動調整欄寬**：標題列右鍵 或 項目右鍵 →「調整欄位至最適大小」
- 底部狀態列：單選顯示完整路徑，多選顯示選取數量

### 操作
- `↑ / ↓` 移動選取，`Enter` 或雙擊開啟
- `F2` 重新命名
- 右鍵選單：開啟 / 重新命名 / 複製路徑 / **開啟路徑（在檔案總管定位該項目）** / 複製 / 剪下 / 貼上 / 刪除
  （在空白處右鍵只會出現「貼上 / 調整欄寬」，避免誤對檔案操作）
- `Ctrl+C / X / V`（清單有焦點時）複製 / 剪下 / 貼上
- `Del` 刪除到資源回收筒（可復原，刪除前會確認）
- 複製／剪下走 **Windows 真剪貼簿**，可與檔案總管互通
- `Esc` 關閉

### 便利功能
- **截圖**：把整個視窗畫面複製到剪貼簿。
- **自訂捷徑**：依視窗寬度自動換行排列；點一下把篩選目錄切過去。**數量可在設定中增減（最少 10 組）。**
- **選資料夾**：手動挑選要篩選的資料夾（Win11 多分頁抓不到當前路徑時的退路）。
- **設定**：可變更啟動熱鍵、設定開機自動載入、增減與編輯捷徑。
- 設定存於腳本同目錄的 `FolderFilter.ini`。

## 安裝與使用

1. 安裝 **[AutoHotkey v2](https://www.autohotkey.com/)**（`winget install AutoHotkey.AutoHotkey`）。
2. 下載 `FolderFilter.ahk`，**雙擊**即可常駐於系統匣。
3. 打開任一資料夾，按 **`Ctrl + Alt + F`**（預設熱鍵，可在「設定」更改）叫出篩選視窗。
4. （選用）開機自動啟動：在「設定」勾選「開機時自動載入」，或把腳本捷徑放進 `shell:startup`。

> 需求：Windows 10/11、AutoHotkey **v2**（v1 不相容）。

---

## 設定檔

`FolderFilter.ini`（與腳本同目錄）會儲存熱鍵與 10 個捷徑。此檔屬個人設定，已加入 `.gitignore`，不會上傳。

---

## 已知限制

- 大型資料夾最多掃描 20 萬筆、排序上限 2 萬筆、清單一次最多顯示 5000 列（達上限時狀態列會提示），縮小關鍵字即可。
- Win11 多分頁的檔案總管，少數情況抓到的當前路徑可能不是作用中的分頁 → 可用「選資料夾」按鈕手動指定。
- 貼上採用基本檔案複製/移動，不含 Explorer 的進度視窗與 ACL/替代資料流等進階語意（規劃中）。

## 開發歷程

這支工具是在一連串迭代中逐步長出來的：

| 階段 | 內容 |
|---|---|
| 雛形 | 熱鍵讀取當前檔案總管路徑 + 即時篩選視窗 |
| 模式與基礎操作 | 篩選／搜尋模式、`↑↓`/Enter/雙擊開啟、Esc 關閉 |
| 進階篩選 | 改為「含子資料夾」勾選；排除關鍵字、行內 `-`、萬用字元 `*.md` |
| 欄位強化 | 新增副檔名、大小、修改日期、建立日期欄，皆可排序 |
| 檔案操作 | F2 改名、右鍵複製路徑／複製／剪下／貼上（CF_HDROP 真剪貼簿）、刪除到資源回收筒 |
| 排除條件 | 排除資料夾／檔案、小於/大於 N KB、N 天前/N 天以後 |
| 便利功能 | 截圖到剪貼簿、強制刷新、10 個自動換行捷徑、欄寬自動調整 |
| 設定系統 | 變更熱鍵、開機自動載入、自訂捷徑，存於 INI |
| 顏色（已移除） | 曾嘗試自訂資料夾／檔案／框線／網底顏色，但自訂文字色會被 Windows 佈景主題覆蓋，整列自繪又會破壞選取顯示，最終回退為系統原生繪製以保選取體驗 |
| **v1.0** | 穩定版：原生繪製、功能完整 |
| **v1.1** | 依 3CLI（Claude/Codex/AGY）審查強化：過濾 debounce、單遍掃描、右鍵先選取游標列、改名/貼上安全驗證、CF_HDROP 記憶體釋放、重開保留狀態、批次錯誤彙總；新增「開啟路徑」選單與「可增減捷徑數量」、「選資料夾」按鈕 |
| **v1.1.1** | 依 v1.1 複審再修：剪貼簿 `GlobalLock` 防護、遞迴模式以相對路徑比對、清單焦點偵測改用 `GetFocus`、部分失敗不清剪貼簿、熱鍵交易式註冊、磁碟根目錄相對路徑修正、純大小寫改名、捷徑上限 30、INI 整段重寫、最小化不存座標、保留欄寬 |

### 技術重點
- 只用 `Shell.Application` COM 讀取當前資料夾路徑，**不注入 explorer.exe**，故對 Windows 更新有韌性。
- 篩選結果在記憶體即時比對，自訂 merge sort 排序，效能良好。
- 檔案複製／剪下使用 Windows 原生 `CF_HDROP` 剪貼簿格式，可與檔案總管雙向互通。
